# 贡献者公约

## 我们的承诺

为了营造一个开放、友好的环境，我们作为贡献者和维护者承诺：无论年龄、体型、身体健全与否、民族、性征、性别认同与表征、经验水平、教育程度、社会地位、国籍、相貌、种族、宗教信仰、性取向，我们都有意为参与者和社区提供没有骚扰的体验。

## 我们的准则

有助于创造积极环境的行为包括但不限于：

* 使用友好和包容性语言
* 尊重不同的观点和经历
* 耐心接受有益批评
* 关注对社区最有利的事情
* 友善对待其他社区成员

不当行为包括但不限于：

* 使用与性有关的言语或是图像，以及不受欢迎的性关注
* 捣乱/煽动/造谣行为，侮辱/贬损的评论，人身攻击及政治攻击
* 公开或私下的骚扰
* 未经许可地发布他人的个人资料，例如住址或是电子地址
* 其他有理由认定为违反职业操守的不当行为

## 我们的责任

项目维护者有责任为「可接受的行为」标准做出诠释，以及对已发生的不当行为采取恰当且公平的纠正措施。

项目维护者有权利和义务去删除、编辑、拒绝违背本行为标准的评论（comments）、提交（commits）、代码、wiki 编辑、问题（issues）等贡献；项目维护者可暂时或永久地封禁任何他们认为行为不当、威胁、冒犯、有害的参与者。

## 适用范围

本行为标准适用于本项目中的所有场合，以及在本项目中代表社区时的公开场合。

代表本项目的场合包括但不限于：使用项目官方电子邮件、通过官方社交媒体账号发布消息、作为指定代表参与在线或线下活动。

## 贯彻落实

可以通过 [项目维护者邮箱] 联系项目维护团队，举报滥用、骚扰或其他不当行为。

维护团队将审议并调查全部投诉，妥善地予以必要的回应。项目团队有义务保密举报者信息。具体执行方针或将另行发布。

未切实遵守或执行本行为标准的项目维护人员，可能会因项目负责人或是其他成员的决定，被暂时或永久地剥夺参与本项目的资格。

## 执行指南

### 如何报告问题

如果您遇到或目睹违反本行为准则的情况，请通过以下方式报告：

1. **发送邮件**: 发送至项目维护者邮箱
2. **GitHub Issues**: 创建私有Issue（仅维护者可见）
3. **直接联系**: 联系您信任的项目维护者

### 报告内容应包括
- 发生时间
- 发生地点（GitHub、邮件列表、会议等）
- 相关人员
- 事件描述
- 相关证据（截图、链接等）

### 处理流程
1. **确认收到**: 维护团队将在48小时内确认收到报告
2. **调查**: 团队将进行调查，可能需要与相关人员沟通
3. **决定**: 基于调查结果做出决定
4. **执行**: 执行相应措施
5. **反馈**: 向举报者反馈处理结果（在保护隐私的前提下）

### 可能的措施
根据违规严重程度，可能采取的措施包括：
- 警告
- 暂时禁止参与
- 永久禁止参与
- 删除不当内容
- 公开道歉要求

## 友好交流指南

### 在GitHub上
- 使用清晰、礼貌的语言
- 避免使用大写字母（被视为喊叫）
- 使用表情符号传达语气
- 假设对方是善意的

### 在代码审查中
- 专注于代码，而不是人
- 使用建设性语言
- 解释为什么需要修改
- 感谢对方的贡献

### 在讨论中
- 一次讨论一个主题
- 引用相关上下文
- 避免离题
- 尊重不同意见
